# Экспериментальные материалы

Эта директория содержит дополнительные и пробные материалы, которые могут быть полезны в реальной работе, но пока не входят в основной процесс репозитория.

Правила:

- не используй `experimental/` как точку входа по умолчанию
- материалы из `experimental/` не должны переопределять `docs/`, `core/`, `gamedev/`, `templates/` или `standards/`
- переносить материал в основной контур стоит только после того, как он доказал полезность на нескольких реальных прогонах

Текущий фокус:

- `browser-gamedev/` для браузерных надстроек, вдохновленных внешним исследованием, без превращения главного `gamedev/` пути в систему только для браузера

## Что это такое

Это справочные заметки и пробные рабочие материалы.
Это не отдельные скиллы и не замена основного процесса репозитория.

Их стоит открывать, когда:

- основной маршрут через `gamedev/` уже понятен
- проект явно ориентирован на браузерную игру
- нужна дополнительная глубина по типу браузерной игры, браузерной отладке или оценочным промтам

Не стоит использовать их как:

- способ обойти `setup-engine`, `map-systems`, `design-system` и другие основные шаги
- замену `core/web-ui-smoke`, `core/web-ui-doctor` или специализированным рекомендациям по рантайму

## Как использовать вместе со скиллами

1. Сначала выбери обычный скилл из `gamedev/`.
2. Открывай подходящую заметку из `experimental/` только тогда, когда нужна именно браузерная глубина.
3. Канонические артефакты проекта все равно веди в обычных местах: `docs/technical-preferences.md`, `design/gdd/`, `reports/`.

Типовые пары:

- выбор стека и рантайма:
  - основной скилл: `gamedev/setup-engine`
  - дополнительная заметка: `experimental/browser-gamedev/archetypes.ru.md`
- карта систем и первый GDD для браузерной игры:
  - основные скиллы: `gamedev/map-systems`, `gamedev/design-system`
  - дополнительная заметка: `experimental/browser-gamedev/archetypes.ru.md`
- браузерная проверка и локальная отладка:
  - основные скиллы: `core/web-ui-smoke`, `core/web-ui-doctor`
  - дополнительная заметка: `experimental/browser-gamedev/browser-debug-checklist.ru.md`
- подготовка оценочных сценариев:
  - основной документ: `docs/gamedev-autoimprovement.md`
  - дополнительная заметка: `experimental/browser-gamedev/eval-prompt-pack.ru.md`

## Короткий маршрут

- Если проект не про браузер, эту директорию можно игнорировать.
- Если проект браузерный, но проблема только в маршрутизации по шагам, оставайся в `gamedev/`.
- Если проект браузерный и нужна дополнительная глубина, используй подходящий файл из `browser-gamedev/`.
