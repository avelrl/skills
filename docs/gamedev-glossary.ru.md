# Gamedev Glossary

Расширенный глоссарий для новичков.
Здесь собраны не только названия шагов, но и базовые термины, которые чаще всего встречаются в `gamedev`.

| Термин | Простое объяснение |
|--------|--------------------|
| `Working / Documentation Language` | Язык, на котором ведутся GDD, отчеты и проектная документация. |
| `Player-Facing Language(s)` | Язык, который видит игрок: меню, HUD, подсказки, субтитры и игровой текст. |
| `Concept` | Краткое описание игры на верхнем уровне: о чем она, для кого она и что в ней главное. |
| `Game Concept` | Конкретный документ `design/gdd/game-concept.md` с описанием идеи игры. |
| `System` | Одна значимая часть игры с понятными границами: например движение, бой, камера или меню. |
| `Systems Index` | Карта всех систем игры и их статусов в `design/gdd/systems-index.md`, то есть общий индекс систем. |
| `GDD` | Документ с правилами одной системы или части игры. Он нужен, чтобы реализация не строилась на догадках. |
| `Pillars` | Несколько главных принципов игры, которые помогают отсекать лишние идеи и принимать спорные решения. |
| `Stack` | Набор основных технологий проекта: движок, язык, библиотека, платформа. |
| `Runtime / Engine` | Среда, в которой игра работает: например Web, Unity, Unreal или Godot. |
| `Scaffold` | Минимальный рабочий каркас проекта: структура папок, точка входа, базовая сборка и запуск. |
| `Artifact` | Конкретный результат шага: документ, отчет, шаблон, индекс систем, каркас проекта. |
| `Prerequisite` | Обязательная предпосылка. Если ее нет, следующий шаг выполнять рано. |
| `Handoff` | Явная передача на следующий шаг или следующему типу работы. |
| `Prototype` | Временная проверка одной рискованной гипотезы до написания рабочего кода. |
| `High-Risk System` | Система, из-за которой может развалиться объем, сроки или архитектура проекта. |
| `MVP` | Минимальная играбельная версия, которая доказывает основную гипотезу игры. |
| `Playable Loop` | Реальный цикл `запуск -> игра -> проигрыш или выход`, который можно пройти за один запуск. |
| `Vertical Slice` | Более узкий, но более отполированный кусок игры, чем грубый MVP. |
| `Demo` | Демо-версия для показа людям. Ее задача — не только доказать, что игра существует, но и ясно ее показать. |
| `Demo Contract` | Явный список того, что демо-версия обязана доказать и показать игроку. |
| `Prepare-Demo` | Шаг, который определяет контракт демо-версии, пробелы, заглушки, риски и следующие шаги. |
| `Onboarding` | То, как игра вводит нового игрока в управление, цель и правила в первые минуты. |
| `HUD` | Постоянные элементы интерфейса во время игры: здоровье, ресурсы, цель, подсказки. |
| `Placeholder` | Временная заглушка вместо финального ассета, текста, экрана или механики. |
| `Asset` | Визуальный, звуковой или другой файл, который используется в игре. |
| `Asset Pipeline` | Правила подготовки, импорта и замены ассетов в проекте. |
| `Playtest` | Реальная проверка игры через запуск и игру, а не только чтение кода или документов. |
| `Smoke Check` | Короткая проверка, что проект хотя бы запускается и не сломан в самых базовых местах. |
| `Tuning` | Подстройка уже существующей игры: числа, темп, окна реакции, читаемость, ритм. |
| `Localization` | Подготовка игры к нескольким языкам и согласование текстов между ними. |
| `Porting` | Перенос игры с одного стека или платформы на другой. |
| `Closure` | Момент, когда шаг или этап можно честно считать закрытым по артефактам и проверкам. |
| `Specialist Overlay` | Платформенно-специфичная глубина поверх общего `gamedev`-процесса, например browser UI или конвейер ассетов. |
