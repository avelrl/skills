# Gamedev Quickstart

Короткий лист с готовыми промтами для самых частых точек входа в `gamedev`.

## Сразу задавай языковую политику

В первом нормальном промте используй примерно такую форму:

```text
Идея игры: ...
Рабочий язык документации: русский.
Язык для игрока: английский и русский.
Текущий этап: концепция.
Выбери и создай следующий правильный артефакт `gamedev`.
```

## Готовые промты

### 0. Есть только концепция, а нужен первый GDD

```text
У меня есть только `design/gdd/game-concept.md`.
Не переходи сразу к `/design-system`.
Сначала используй `/setup-engine`.
Потом используй `/map-systems`.
После этого предложи, для какой системы первым вызвать `/design-system`.
Рабочий язык документации: русский.
Язык для игрока: английский.
```

Ожидаемый маршрут: `/setup-engine` -> `/map-systems` -> первый `/design-system`

### 1. Есть только идея

```text
Создай `design/gdd/game-concept.md` для небольшой динамичной игры с видом сверху.
Рабочий язык документации: русский.
Язык для игрока: английский.
После этого выполни только ближайший правильный шаг `gamedev`.
```

Ожидаемый старт: `/setup-engine` после того как концепция уже существует.

### 2. Не знаю стек

```text
Используй `/setup-engine`.
Подбери стек для этой игры и обнови `docs/technical-preferences.md`.
Рабочий язык документации: русский.
Язык для игрока: английский.
Целевые платформы: сначала Web, позже Unity как отдельная линия портирования.
```

Ожидаемый старт: `/setup-engine`

### 3. Нужна карта систем

```text
Используй `/map-systems`.
Разбей эту игру на системы и обнови `design/gdd/systems-index.md`.
Веди документацию на русском языке.
Термины для игрока держи согласованными с английским текстом интерфейса.
```

Ожидаемый старт: `/map-systems`

### 4. Нужен один GDD для системы

```text
Используй `/design-system combat-loop`.
Напиши GDD боевой системы на русском языке.
Язык для игрока: английский.
Используй существующий индекс систем и не переходи к написанию кода.
```

Ожидаемый старт: `/design-system`

### 5. Нужно проверить одну рискованную механику

```text
Используй `/prototype`.
Проверь механику рывка и оцени, остается ли боевой цикл понятным для игрока.
Не трогай рабочий код.
Оформи результат как отчет о прототипе.
```

Ожидаемый старт: `/prototype`

### 6. Нужен первый играбельный срез

```text
Используй `/assemble-mvp`.
Собери минимальный реальный игровой цикл из уже реализованных систем.
Не расширяй объем работы.
Подготовь отчет по сборке MVP и потом предложи следующий шаг.
```

Ожидаемый старт: `/assemble-mvp`

### 7. MVP уже работает, теперь нужна нормальная демо-версия

```text
Используй `/prepare-demo`.
Подготовь первую демо-версию для показа другим людям на основе текущего играбельного среза.
Это не запрос только на UI.
Определи критичные пробелы по игровому процессу, первым подсказкам, HUD, меню, временным заглушкам, ассетам и проверке качества.
Подготовь `reports/demo-readiness.md` и предложи следующий шаг.
```

Ожидаемый старт: `/prepare-demo`

### 8. После MVP развалился UI

```text
Используй `/prepare-demo`, если по состоянию проекта это ближайший правильный шаг.
Основной игровой цикл уже есть, но слой подачи не готов к демо.
Выполни ближайший правильный шаг процесса.
Не прыгай сразу в случайную реализацию интерфейса.
```

Ожидаемый старт: обычно `/prepare-demo`
